# 已知问题和待确认项

1. 真实内部 API 的路径、鉴权头、请求/响应字段和错误码尚未提供；`HttpBackend` 使用可替换的约定路径。
2. VN 区域准确的运单号规则尚未确认；Demo 兼容原 DSL：`JT` + 8–13 位数字或 8–15 位纯数字。
3. 包裹体积查询不在原 DSL 中；Mock 返回长宽高与体积重，真实计费公式待确认。
4. 签收未收到的身份校验规则不在 DSL 中；Demo 使用手机号后四位，真实权限仍必须由业务接口决定。
5. 投诉类型、证据材料、理赔金额和 SLA 未提供；Demo 仅创建调查/投诉预受理单，不承诺结果。
6. 改址支持范围、费用、跨区域限制和敏感地址策略未提供；Demo 只允许运输中/延误状态尝试。
7. FAQ 文本来自 DSL 内嵌示例，不是正式 VN 政策库。
8. 生产 Checkpointer、Redis/PostgreSQL 连接方式、会话 TTL 和数据保留期尚未确定。
9. 当前 API 没有 SSE；如增加流式输出，应在状态提交后再结束事件流。
10. 模型 Token 用量依赖具体 OpenAI-compatible 供应商返回格式，当前只预留模型耗时日志。
